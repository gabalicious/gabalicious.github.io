Active Record Jan 4

What is Active Record

Active Record is an API which is part of the Ruby on Rails framework. It is the model in the 'MVC' system. Active Record is responsible for all the business data and logic that is saved to the database.

Why is it so awesome?

Active Record allows rails developers to program all the business logic using Ruby without the need for SQL and works with almost all SQL relational databases. This means that Active Record can use one piece of code and it can work for different SQL Databases. Behind the scenes the API is making SQL queries on the database. Active Record uses the proper SQL query for whichever SQL database you choose. (Example: You may have a SQLite database for your UAT environment but a Oracle Database for Production. Active Record will use the correct queries for each environment without the need for duplicative code. )

What else can it do?

It can also help with assigning many to many or one to one relationships.

Any Negatives?

The only negative is that it requires you to read documentation about the Active Record API in Rails. The good news is if you want to use SQL queries you can still do so.

Procs

A Proc or procedure allows you to make an object and call it later by using the .call method. Procs Code

square = Proc.new{|n| n * n }
multiply = Proc.new {|x, y| x * y }

p square.call(3)              # or you may use this syntax square[3] 
p multiply.call(4, 3)      # or you may use this syntax multiply[4, 3]

As you can see, in the first call square.call(3). The number three is the parameter that is defined as 'n' in the square method. But in the multiply method there are two parameters, 4 and 3 which are x and y in the method.

Lambdas

Both procs and Lambdas are kinds of Proc classes. The only difference is how they return an object. I found a post here that has more info on their differences. Lambdas Code

sentence = lambda {puts "Hello Boys and Girls."} # same as sentence = proc {puts "Hello Boys and Girls."}
sentence.call
puts "Is sentence a lambda - #{sentence.lambda?}"

Much like the Procs code, in the Lambdas Code, the method is called using .call.

Ruby Classes 101 Nov 30

Best Use Cases for Classes

The best way to use Classes is when working with data whose methods repeat and have similar data structure. Let's say we had a Sports statistics website. We can have a class that represents a player. Each player needs to be able to be able to have data about them stored (Height, age, stats, picture, rating) and also be able to have different methods like one that prints a graphical card image of the player.

class Player
    attr_accessor :height, :dob, :stats, :picture, :rating, :name
    def initialize(name,height,dob,rating)
        @name = name
        @height = height
        @dob = dob
        @stats = {}
        @rating = rating
    end

    def printInfo
        puts "name: #{@name}"
        puts "height: #{@height}"
        puts "dob: #{@dob}"
        puts "rating: #{@rating}"
    end
end

Including Our Class File

The classes.rb file above is a good example of when to use a classes. The initialize method stores info stores default info about each player. The printInfo method prints to the console some info about that player. Let's make a new Player and and then use the printInfo method to print player info.

require_relative 'classes.rb'
p1 = Player.new("Rubin TI Jones","5'11","01-13-1989","101") # (name,height,dob,rating)
p1.rating ="105"
p1.printInfo

The expected Output will be:

name: Rubin TI
height: 5'11
dob: 01-13-1989
rating: 101
