Git 101

Posted on: 11/03/2014 Read Time: 5 Minutes

Git is a very popular version control system used to manage different versions of code. It was designed and developed by Linus Torvalds in 2005 so he could manage the all the different versions of the Linux Kernel with the open source community. Git allows developers to track old versions of code, existing code, and new features all in directory by using the power of branches. Each Branch can store it's own unique version of the code.

To use Git, first the developer must initiate a Git repository by typing 'git init' in the terminal under a directory of a project. This will create a hidden folder called '.git'. Git will track all the changes that are made in this hidden folder.

GitHub is a GUI version of Git on the web with some additional features. One cool feature is the ability to comment on changes that were made. GitHub has become the go to place for the open source developer community to share and collaborate with one another on code. Many companies even request to see job applicant's GitHub pages often instead of a resumes.
